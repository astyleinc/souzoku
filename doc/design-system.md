# Ouver デザインシステム

**作成日: 2026年4月16日**
**ステータス: Phase 1 初期定義**

---

## 1. デザイン原則

### 1.1 ブランドポジション

Ouverは相続不動産の専門プラットフォーム。一般的な不動産ポータル（SUUMO, HOME'S）の「楽しい住まい探し」ではなく、**相続という課題を信頼できる仕組みで解決する**サービス。

| 軸 | 方向性 |
|---|---|
| トーン | カジュアル ← **ここ** → フォーマル |
| 情報密度 | 詰め込み ← → **ここ** ゆとり |
| 配色 | 派手 ← → **ここ** 落ち着き |
| 類似ジャンル | リーガルテック、金融系SaaS |

### 1.2 4つの原則

1. **信頼が最優先** — 色・余白・タイポグラフィすべてで「安心して任せられる」印象を作る
2. **数字は明瞭に** — 価格・入札数・報酬額など、金額が意思決定の中心。数字の視認性に妥協しない
3. **ステータスは一目で** — 物件・入札・案件のステータスが多い。色×形で直感的に判別できる体系を持つ
4. **ロールに合わせて削ぐ** — 5つのロールそれぞれに必要な情報だけを見せる。全部入りの画面は作らない

## 2. カラーパレット

### 2.1 色の役割設計

| 役割 | 用途 | 他の役割に使わない |
|------|------|----|
| **Brand色** (Primary/Secondary) | ロゴ、ナビ、見出し、選択状態 | CTAボタンには使わない |
| **操作色** (CTA) | 主要アクションボタン、リンク | 状態表示には使わない |
| **状態色** (Success/Warning/Error/Info) | ステータスバッジ、アラート | ナビやブランド要素には使わない |
| **注意色** (Error/Warning) | フォームエラー、期限切れ | 装飾には使わない |

## 3. タイポグラフィ

### 3.1 フォント

| 用途 | フォント | 理由 |
|------|---------|------|
| 日本語本文 | Noto Sans JP | 可読性、ウェイトの豊富さ |
| 英数字・価格 | Inter | tabular-nums対応、数字の視認性 |
| コード・ID | JetBrains Mono | 等幅で誤読防止 |

### 3.2 サイズ階層

| 名前 | サイズ | ウェイト | 用途 |
|------|--------|---------|------|
| Display | 32px (2rem) | Bold | ヒーローのみ |
| H1 | 24px (1.5rem) | Bold | ページタイトル |
| H2 | 20px (1.25rem) | SemiBold | セクション見出し |
| H3 | 16px (1rem) | SemiBold | カード見出し |
| Body | 14px (0.875rem) | Regular | 本文 |
| Caption | 12px (0.75rem) | Regular | 補足・注釈・タイムスタンプ |
| Price Large | 28px (1.75rem) | Bold / Inter | 物件一覧の価格 |
| Price Medium | 24px (1.5rem) | Bold / Inter | 入札画面の価格 |

### 3.3 行間

- 本文: `line-height: 1.7`（日本語に最適化）
- 見出し: `line-height: 1.3`
- 価格: `line-height: 1.0`

## 4. コンポーネントパターン

### 4.1 ステータスバッジ

色×スタイルで体系化。**同じステータスには必ず同じ色を使う。**

#### 物件ステータス

| ステータス | 色 | スタイル | class例 |
|-----------|----|----|-----|
| 審査待ち | Neutral | Outline | `bg-neutral-50 text-neutral-600 border-neutral-300` |
| 公開 | Primary | Solid | `bg-primary-500 text-white` |
| 公開（登記中） | Warning | Outline | `bg-warning-50 text-warning-700 border-warning-500` |
| 入札受付中 | CTA | Solid | `bg-cta-500 text-white` |
| 入札終了 | Neutral | Solid | `bg-neutral-400 text-white` |
| 成約 | Success | Solid | `bg-success-500 text-white` |
| 差戻し | Error | Outline | `bg-error-50 text-error-700 border-error-500` |
| 不成立 | Neutral | Outline | `bg-neutral-50 text-neutral-500 border-neutral-300` |

#### ルール

- **Solid** = 確定状態（公開、成約、入札受付中）
- **Outline** = 暫定・要対応状態（審査待ち、差戻し、登記中）
- 同じ意味には同じ色: 「成約」は物件でも入札でも案件でも常に `Success / Solid`

### 4.2 物件カード

```
┌──────────────────────────────────┐
│  ┌────────────┐  ┌───────────┐  │
│  │            │  │ 入札受付中  │  │  ← ステータスバッジ（右上）
│  │   写真     │  └───────────┘  │
│  │  4:3比率   │                 │
│  │            │  3,500 万円     │  ← .price クラス（Inter, Bold）
│  └────────────┘                 │
│                                 │
│  東京都練馬区豊玉北              │  ← 所在地（1行、溢れたら省略）
│  マンション・72.5㎡・築23年      │  ← スペック（区切り: ・）
│                                 │
│  ┌──────┐  入札 5件             │  ← 緊急度バッジ + 入札数
│  │3ヶ月  │                      │
│  └──────┘                       │
└──────────────────────────────────┘
```

- 写真なしの場合: プレースホルダー（`neutral-100`背景 + 建物アイコン）
- 価格は常に「万円」表記（桁区切りカンマあり）
- 所在地は市区町村まで。番地は詳細ページ

### 4.3 ボタン

| 種類 | 用途 | スタイル |
|------|------|---------|
| Primary | 入札する、登録する、公開する | `bg-cta-500` + white text |
| Secondary | キャンセル、戻る | `border-neutral-300` + neutral text |
| Ghost | フィルターリセット、もっと見る | テキストのみ + underline |
| Danger | 案件中止、削除 | `bg-error-500` + white text |

- 角丸: `radius-md` (8px)
- 高さ: 40px (通常) / 48px (大きなCTA)
- disabled: `opacity-50 cursor-not-allowed`

### 4.4 フォーム

- ラベル: `text-caption` + `text-neutral-600` + 上部配置
- 入力: `border-neutral-200` / focus: `ring-primary-500`
- エラー: `border-error-500` + `text-error-500` のヘルプテキスト
- 必須マーク: `*` を `text-error-500` で表示

### 4.5 テーブル（管理画面）

- ヘッダー: `bg-neutral-50` + `text-neutral-600` + `font-semibold`
- 行: hover → `bg-neutral-50`
- ボーダー: 行間のみ `border-b border-neutral-200`
- ページネーション: テーブル下部、中央揃え

## 5. レイアウト

### 5.1 ブレークポイント

| 名前 | 幅 | 用途 |
|------|------|------|
| sm | 640px | モバイル横 |
| md | 768px | タブレット |
| lg | 1024px | デスクトップ |
| xl | 1280px | ワイドデスクトップ |

### 5.2 コンテンツ幅

- 公開ページ（トップ、物件一覧）: `max-w-7xl` (1280px)
- 物件詳細: `max-w-4xl` (896px)
- ダッシュボード: サイドナビ 240px + 残り全幅
- フォーム: `max-w-lg` (512px)

### 5.3 間隔の基本単位

Tailwindのデフォルトスケール（4px基準）を使用。よく使う間隔:

| 用途 | サイズ | Tailwind |
|------|--------|---------|
| カード内パディング | 16px | `p-4` |
| セクション間 | 48px | `py-12` |
| カード間ギャップ | 24px | `gap-6` |
| フォーム項目間 | 16px | `space-y-4` |
| インラインアイコン横 | 8px | `gap-2` |
