## 1. 全体概要

### 1.1 システムの目的

相続で不動産を取得した売主が、士業（税理士・司法書士等）の紹介を通じて物件を掲載し、入札方式により適正価格・短期間で売却できるポータルプラットフォーム。

### 1.2 システムドメイン構成

| # | ドメイン | 主な利用者 |
|---|---------|-----------|
| 1 | 物件ポータル | 売主・買い手・一般閲覧者 |
| 2 | 入札システム | 買い手・売主・提携業者 |
| 3 | 士業パートナー管理 | 税理士・司法書士・Ouver運営 |
| 4 | 業者連携・案件管理 | 提携宅建業者・Ouver運営 |
| 5 | 収益管理 | Ouver運営・経理 |
| 6 | 管理ダッシュボード | Ouver運営 |

### 1.3 三者ビジネスモデルとシステムの関係

```
士業パートナー ──紹介──→ 売主 ──物件登録──→ Ouverプラットフォーム
                                                    │
                                              買い手が入札
                                                    │
                                              成約（運営承認）
                                                    │
                                            提携宅建業者が仲介実務
                                            （問い合わせ・内見・重説・契約・決済）
                                                    │
                                              決済完了
                                                    │
                                              報酬配分処理
```

**士業の認証フロー：**
```
士業が自己登録（資格登録番号を入力）
  ↓
システムが番号の形式チェック（自動）
  ↓
スクレイピングで士業会サイトを自動照合
  ↓
照合OK → ステータス「認証済み」
照合NG → Ouver運営に通知 → 手動確認
  ↓
確認OK → 「認証済み」
確認NG → アカウント停止 + 本人に連絡
```

**NWと所属事務所の関係：**
```
士業（個人）
  ├── 所属事務所（勤務先。独立・移籍で変更あり）
  └── 所属NW（コミュニティ。複数所属可。脱退あり）

※ NW ≠ 所属事務所
※ NW無しの事務所からの直接紹介もあり得る
```

**変更履歴を残す項目：**
- 所属事務所（独立・移籍の記録）
- 所属NW（脱退後の報酬配分判定に必要）
- 報酬支払先（支払いトラブル防止）
- 就業形態（会社員→独立の記録）

## 3. ドメイン1: 物件ポータル

### 3.1 物件登録の流入パターン

| パターン | 流れ |
|---------|------|
| A. 売主が自分で登録 | 売主がサイトで直接登録 |
| B. 士業が売主に案内→売主が登録 | 士業の紹介リンク経由で売主が登録。紹介者が自動紐づけ |
| C. 士業（or 秘書）が代理登録 | 士業アカウントから売主に代わって物件情報を入力 |

### 3.2 紹介コード付きURL

```
紹介リンクの例:
https://souzoku-matching.vercel.app/register?ref=TAX_TANAKA_001

→ 売主がこのリンクから登録すると、自動的に紹介士業として記録
→ 士業が複数NWに所属している場合、「この紹介はどのNW経由か」を選択
→ NW経由でない場合は「直接紹介」を選択可
```

### 3.3 最低出品価格

- **1,000万円未満は登録不可**（バリデーション）
- 価値のない物件は1,000万円以上で買われないため、市場原理に任せる

### 3.4 必要書類

| 書類 | 必須/任意 | 備考 |
|------|----------|------|
| 登記事項証明書（登記簿謄本） | 必須 | 所有者・権利関係の確認 |
| 遺産分割協議書 or 遺言書 | 必須 | 売却権限の証明。相続登記済みなら不要 |
| 相続登記完了の証明 | 任意 | 未了でも出品可（「登記手続き中」表示） |
| 固定資産税納税通知書 | 任意 | 評価額確認 |
| 本人確認書類 | 必須 | 売主の身元確認 |
| 物件の写真 | 必須 | 掲載用 |

### 3.5 物件公開フロー（書類審査）

```
売主が物件登録
  ↓
ステータス: 「審査待ち」
  ↓
Ouver運営が必要書類の確認（内容は問わない、揃っているかだけ）
  ↓
├→ 書類不足 → ステータス: 「差戻し」（不足書類を通知）
│              → 売主に通知、修正を依頼
│
└→ 書類揃い → ステータス: 「公開」+ 提携業者を割り当て
               → 売主・紹介士業に通知
```

### 3.6 相続登記未了物件の管理

```
出品（登記未了）
  ↓
ステータス: 「公開（登記手続き中）」
  ↓
├→ 2週間ごと: 「登記手続きの進捗はいかがですか？」通知（メール）
│
├→ 登記完了書類がアップロードされた → 「公開」に切替
│
└→ 2ヶ月経過しても登記書類なし → 「差戻し」
    → 売主に通知:「登記完了後に再出品できます」
```

### 3.7 書類の閲覧権限

- 売主がアップロードした書類は、**売主が士業ごとに閲覧許可を出す**
- 紹介した士業が自動的に見れるわけではない（売主がコントロール）
- Ouver運営・提携業者は業務上閲覧可能

### 3.8 機能一覧

| 機能 | 内容 | 既存/新規 |
|------|------|----------|
| 物件登録（売主） | 売主がフォームから登録 | 既存 |
| 最低出品価格 | 1,000万円未満は登録不可（バリデーション） | 新規 |
| 必要書類アップロード | 登記簿謄本・本人確認書類・写真等 | 要改修 |
| 書類審査 | 必要書類が揃っているかの確認のみ。内容は問わない | 新規 |
| 審査ステータス | 審査待ち → 公開 / 差戻し | 新規 |
| 登記未了の出品 | 「登記手続き中」表示で公開可 | 新規 |
| 登記催促通知 | 2週間ごとにメール通知 | 新規 |
| 登記未了の自動差戻し | 2ヶ月経過で差戻し、再出品可 | 新規 |
| 紹介コード付きURL | 士業ごとに発行、売主登録時に自動紐づけ | 新規 |
| 代理登録 | 士業/秘書が売主に代わって物件登録 | 新規 |
| 士業への進捗通知 | 登録・公開・入札・成約時にメール+システム内通知 | 新規 |
| 物件検索・一覧 | フィルター・ソート・ページネーション | 既存 |
| 物件詳細ページ | 画像ギャラリー・概要・関連物件 | 既存 |
| SEOページ | エリア別・種別別ランディングページ・サイトマップ | 既存 |

## 4. ドメイン2: 入札システム

### 4.1 入札の基本ルール

| 項目 | 仕様 |
|------|------|
| 希望価格（= 最低入札価格） | 売主が設定。これ未満の入札は不可 |
| 即決価格 | 売主が任意で設定。到達時に売主+士業に通知 |
| 入札期間 | 売主が自由に設定 |
| 入札期間の変更 | 1回のみ可。2回目以降は一度終了して再出品 |
| 入札回数 | 1人何度でも更新可能 |
| 入札内容の公開 | 他の入札者の金額は見えない（sealed bid） |
| 入札の閲覧権限 | 売主・紹介士業・提携業者・Ouver運営のみ |

### 4.2 入札〜成約のフロー

```
買い手が入札（期間内なら何度でも更新可）
  │
  ├→【通常】入札期間終了
  │   → 売主に全入札一覧を提示（金額順）
  │   → 売主が入札者を選択
  │
  ├→【即決】即決価格以上の入札が入った
  │   → 売主+士業に通知（承認期限48時間）
  │   → 売主が承認 or 見送り
  │   → 見送りの場合、入札期間終了まで継続
  │   → 48時間期限切れの場合、「見送り」扱いで継続
  │
  ▼
売主が入札者を選択
  │
  │ 最高額が原則。他を選ぶ場合は理由を選択式で記録:
  │   ・与信/資金調達に不安がある
  │   ・取引条件（引渡時期等）が合わない
  │   ・その他（自由記述）
  │
  ▼
ステータス: 「承認待ち」
  │ Ouver運営が確認
  │  ・最高額以外を選んだ場合の理由は妥当か
  │  ・不正な取引ではないか
  │
  ├→ 問題なし → ステータス: 「成約」
  │              → 全関係者に通知（売主・買い手・士業・提携業者）
  │              → 提携宅建業者の仲介実務に移行
  │
  └→ 問題あり → ステータス: 「差戻し」
                 → 売主に理由を通知、再選択を依頼
```

### 4.3 入札不成立の場合

入札が1件も入らなかった、または全入札を売主が辞退した場合、売主が選択：
- **通常掲載に切り替え**：入札は終了、問い合わせベースで買い手を募集
- **再出品**：価格を見直して再度入札を設定

### 4.4 入札のステータス一覧

| ステータス | 意味 |
|-----------|------|
| 入札受付中 | 入札期間内。買い手が入札可能 |
| 入札終了 | 期間終了。売主の選択待ち |
| 承認待ち | 売主が選択済み。Ouver運営の確認待ち |
| 成約 | Ouver運営が承認。提携業者の仲介に移行 |
| 差戻し | 運営が問題ありと判断。売主に再選択依頼 |
| 不成立 | 入札ゼロで期間終了 / 売主が全入札を辞退 |

### 4.5 機能一覧

| 機能 | 内容 | 既存/新規 |
|------|------|----------|
| 希望価格（= 最低入札価格） | これ未満の入札は不可 | 既存 |
| 即決価格 | 売主が任意で設定。到達時に売主+士業に通知、承認期限48時間 | 新規 |
| 入札期間 | 売主が自由に設定。変更は1回のみ可。2回目以降は再出品 | 要改修 |
| 入札回数 | 1人何度でも更新可能 | 要改修 |
| 入札の非公開 | 他の入札者の金額は見えない。売主・紹介士業・提携業者・Ouverのみ閲覧可 | 既存 |
| 入札終了後の選択 | 最高額が原則。他を選ぶ場合は理由を選択式で記録 | 新規 |
| 承認待ちステータス | 売主が選択後、Ouver運営が最終確認して成約 | 新規 |
| 運営の差戻し | 問題ありの場合、売主に理由通知+再選択依頼 | 新規 |
| 不成立時の選択肢 | 「通常掲載に切替」or「再出品」を売主が選択 | 新規 |
| 通知 | 入札あり・即決到達・期間終了・成約をメール+システム内通知 | 新規 |

## 5. ドメイン3: 士業パートナー管理

### 5.1 NWの位置づけ

- NW会社（awaka cross / UIコンサルティング / ミツカル等）は**代理店的な特別枠**
- 士業をOuverに紹介してくれる組織
- NW ≠ 所属事務所（別の概念）
- 士業はNWに複数同時所属可能、脱退もあり得る
- NW無しの事務所からの直接紹介もあり得る

### 5.2 紹介時のNW選択フロー

```
士業が売主を紹介（紹介リンク or 代理登録）
  ↓
士業がNWを複数持っている場合
  → 「この紹介はどのNW経由ですか？」を選択
     ・awaka cross
     ・UIコンサルティング
     ・ミツカル
     ・NW経由ではない（直接紹介）
  ↓
案件に紐づけて記録
```

### 5.3 報酬配分ルール

| パターン | 業者 | Ouver | 士業 | NW |
|---------|------|-------|------|-----|
| NW経由の紹介 | 50% | 32% | 15% | 3% |
| NW無し（直接紹介） | 50% | 35% | 15% | 0% |
| NW脱退後の新規紹介 | 50% | 35% | 15% | 0% |
| NW脱退前に紹介済みの案件が成約 | 50% | 32% | 15% | 3% |

※ 報酬配分は**案件単位で「NW経由か否か」を記録**して判定
※ NW脱退後でも、脱退前に紹介された案件の成約はNWに支払い
※ Phase 1は士業の紹介料率は一律15%。Phase 2で実績に応じた個別設定

### 5.4 士業マイページ機能

| 機能 | 内容 | Phase |
|------|------|-------|
| プロフィール管理 | 個人情報・就業形態・事務所・NW・報酬設定の編集。変更履歴を自動記録 | Phase 1 |
| 紹介リンク発行 | 自分の紹介コード付きURLの確認・コピー | Phase 1 |
| 紹介時のNW選択 | 複数NW所属の場合、どのNW経由か選択。「NW経由でない」も選択可 | Phase 1 |
| 代理登録 | 売主に代わって物件情報を入力 | Phase 1 |
| 紹介案件一覧 | 紹介した売主の物件リスト+ステータス | Phase 1 |
| 書類閲覧 | 売主が閲覧許可を出した書類のみ閲覧可 | Phase 1 |
| 紹介料の実績 | 成約案件の紹介料金額・支払いステータス | Phase 1 |
| 通知 | 物件登録・公開・入札・即決到達・成約時にメール+システム内通知 | Phase 1 |
| 紹介料率の個別設定 | 実績に応じた料率変更 | Phase 2 |

### 5.5 Ouver管理画面側の機能

| 機能 | 内容 | Phase |
|------|------|-------|
| 士業一覧・検索 | 資格種別・NW・ステータスで絞り込み | Phase 1 |
| 認証管理 | 資格番号の自動照合（スクレイピング）結果確認、手動での認証ステータス変更 | Phase 1 |
| 代理登録 | IT苦手な士業のアカウントを運営が作成 | Phase 1 |

## 6. ドメイン4: 業者連携・案件管理

### 6.1 案件フロー（全体）

```
物件公開時に提携業者を割り当て（Ouver運営が手動）
  ↓
買い手から問い合わせ・内見予約 → 業者が対応
  ↓
入札 → 成約（運営承認）
  ↓
重説 → 契約 → 決済
  ↓
決済完了書類アップロード → 報酬配分処理へ
```

※ 業者の割り当ては**物件公開時点**で行う（入札前から問い合わせ・内見対応が必要なため）

### 6.2 業者名の非公開・直契約防止

| 対策 | 内容 |
|------|------|
| 業者名の非公開 | 物件ページには「提携宅建業者が対応」とだけ表示。業者名・連絡先は出さない |
| 連絡はシステム経由 | 問い合わせ・内見予約はOuverのシステムを通す |
| 契約で縛る | 業務委託契約に直契約禁止条項+違約金を明記 |

### 6.3 問い合わせ・内見予約の仕組み

**Phase 1はシステムは窓口のみ、メール通知で連携（チャット機能は不要）：**

```
買い手が問い合わせフォーム送信
  ↓
Ouverシステムが業者にメール通知
「新しい問い合わせがあります。システムから確認してください」
  ↓
業者がシステムにログインして返信
  ↓
買い手にメール通知
「回答がありました。システムから確認してください」
```

- やり取りの記録は案件ごとのスレッド形式でシステムに保存
- 業者の直接の連絡先は買い手に渡らない

### 6.4 案件ステータス管理

| ステータス | 更新者 | アップロード書類 | 通知先 |
|-----------|--------|----------------|--------|
| 業者割当済み | Ouver運営 | - | 業者・売主・買い手・士業 |
| 売主連絡済み | 業者 | - | - |
| 買い手連絡済み | 業者 | - | - |
| 重説完了 | 業者 | 重要事項説明書 | Ouver運営 |
| 契約締結 | 業者 | 売買契約書 | Ouver運営・士業 |
| 決済完了 | 業者 | 決済完了証明書類 | Ouver運営・売主・士業 → 報酬配分処理へ |
| 案件中止 | 業者 or 運営 | - | 全関係者 |

### 6.5 業者評価

|------|------|-------|

## 7. ドメイン5: 収益管理

### 7.1 お金の流れ

```
売主・買主
  │ 仲介手数料（例: 102万円）
  ▼
提携宅建業者
  │ 自社分を引いて残りをOuverに振込
  │  （例: 51万円を引いて、51万円をOuverへ）
  ▼
Ouver
  ├→ 士業に紹介料を振込（例: 15.3万円）
  ├→ NW会社にNW利用料を振込（例: 3.1万円）
  └→ Ouverの手元に残る（例: 32.6万円）
```

※ Ouverが全体のハブとなり、士業・NWへの支払いを一括管理（トラブル防止）

### 7.2 配分計算の例（1,500万円物件・NW経由）

```
成約価格:           1,500万円
仲介手数料（両手）:  1,500万 × 3% + 6万 = 51万 × 2 = 102万円（税別）

提携業者（50%）:    51.0万円
Ouver（32%）:       32.6万円
士業（15%）:        15.3万円
NW（3%）:            3.1万円
合計:              102.0万円
```

### 7.3 段階的手数料率（業者との交渉用）

| フェーズ | 業者 | Ouver | 士業 | NW | 条件 |
|---------|------|-------|------|-----|------|
| 最初の5件 | 60% | 22% | 15% | 3% | 実績作り期間 |
| 6〜20件 | 55% | 27% | 15% | 3% | 安定供給の証明後 |
| 21件〜 | 50% | 32% | 15% | 3% | 本来条件に移行 |

### 7.4 片手仲介への対応

買い手が自分の仲介業者を連れてきた場合（片手仲介）：
- 業者の取り分を**60%**に引き上げる補正条項
- Ouverの買い手プールから入札させることで**両手仲介を基本設計**とする

### 10.1 買い手ランクシステム

```
ランク判定要素:
  ├── 購入実績数

  ├── 築年数・面積の補正

  ├── 無料会員: 見れない
  └── サブスク会員: 見れる

表示例:
  「この物件の推定相場: 1,800万円」
  「希望価格1,500万円 → 相場より約17%お得」
```

仲介手数料の配分だけでなく、買い手からのサブスク収入を追加。

LINE公式アカウント + Messaging APIで全ユーザーに通知対応。

### 10.5 自動振込（Stripe Connect）

月額固定費0円。振込1件あたり250円 + 振込金額の0.25%。
月8件成約（32回振込）で月間コスト約2万円。

### 10.6 業者自動割り当て

評価スコア × 対応エリアに基づく自動振り分け。

### 10.7 買い手からの業者評価

売主に加え、買い手からも成約後に業者評価を収集。

## 付録

### A. ステータス遷移図（物件）

```
売主が登録
  → 審査待ち
    → 公開（書類OK）
      → 公開（登記手続き中）※登記未了の場合
        → 差戻し（2ヶ月経過で自動）
        → 公開（登記完了書類アップで切替）
      → 入札受付中（入札ON）
        → 入札終了
          → 承認待ち（売主が選択）
            → 成約（運営承認）
              → 業者割当済み → 重説完了 → 契約締結 → 決済完了
            → 差戻し（運営が問題あり判断）
          → 不成立
            → 通常掲載に切替
            → 再出品
      → 通常掲載（入札OFF・問い合わせベース）
    → 差戻し（書類不足）
      → 審査待ち（売主が修正・再提出）
```
